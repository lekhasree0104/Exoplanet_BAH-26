# 🪐 AI-Enabled Exoplanet Detection Pipeline
### ISRO Hackathon — Problem Statement: AI-driven Detection of Exoplanets from Noisy Astronomical Light Curves

---

## Overview

This project is a full end-to-end AI-powered pipeline for detecting exoplanet transit signals in noisy astronomical light curves from NASA's TESS mission. It automatically classifies periodic dips in stellar brightness as planet transits, eclipsing binaries, starspots, or contaminated blends, and fits physical transit models to estimate planet parameters.

**What makes it AI-driven:**
- A **Random Forest classifier** trained on real labeled TESS light curves (from ExoFOP) classifies each detected signal across 4 categories
- **17 physically-motivated features** extracted per signal, including stacked SNR, FAP proxy, secondary eclipse ratio, odd-even depth asymmetry, and sinusoidal correlation
- A science dashboard built on **FastAPI** delivers live analysis on any star with a single search

---

## Pipeline Architecture

```
NASA MAST Archive (lightkurve)
         ↓
  Download light curve
         ↓
  Detrend (Savitzky-Golay flatten)
         ↓
  BLS Periodogram — up to 3 independent signals
  (mask-and-repeat for multi-planet detection)
         ↓
  Extract 17 features per signal
         ↓
  Random Forest classifier → planet / EB / starspot / blend
         ↓
  Stacked SNR + FAP proxy for significance
         ↓
  Batman transit model fit → Rp, a/Rs, habitable zone
         ↓
  Science dashboard + batch ranked output
```

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/<your-repo>.git
cd <your-repo>
```

### 2. Install dependencies

```bash
pip install fastapi uvicorn lightkurve astropy numpy pandas scipy scikit-learn matplotlib batman-package groq requests
```

> **batman-package** is required for transit model fitting. If it fails to install on Windows, try:
> ```bash
> pip install batman-package --no-build-isolation
> ```

### 3. Set your Groq API key (optional — only needed for `/chat` endpoint)

```bash
# Windows
set GROQ_API_KEY=your_key_here

# Linux / Mac
export GROQ_API_KEY=your_key_here
```

The dashboard's **single-star analysis** (`/analyse`) works **without** a Groq key. The key is only needed if you use the `/chat` agent endpoint.

## Running the Pipeline

### Step 1 — Build real training data (run once)

Downloads real labeled TESS light curves from ExoFOP and MAST, extracts 17 features from each, and saves to `training_features.csv`.

```bash
python build_real_training_data.py
```

This takes **15–20 minutes** (downloads ~240 light curves from NASA MAST). You will see:

```
Fetching TOI table...
  Got 8064 TOIs from ExoFOP CSV.
  [planet            ] TIC  275421144  (1/60) ... OK
  ...
Saved 240 rows to training_features.csv
Class breakdown:
  planet              : 60
  eclipsing_binary    : 60
  starspot            : 60
  blend               : 60
```

> If some classes hit download failures, the script automatically fills gaps with realistic synthetic data.

---

### Step 2 — Start the server

```bash
uvicorn main:app --reload
```

On first run, the server trains a temporary classifier on synthetic data while it starts up. You will see:

```
[clf] Training on synthetic data (600 per class)...
[clf] Trained and saved.
INFO: Application startup complete.
```

Open your browser at **http://localhost:8000**

---

### Step 3 — Load real training data into the classifier

In the dashboard sidebar, find **"Load Real Training Data"**, make sure the path says `training_features.csv`, and click **LOAD**.

You should see: `✓ Retrained on 240 samples`

From this point the classifier has been trained on real TESS data and `classifier.pkl` is saved to disk. Future server restarts will load it instantly without retraining.

---

### Step 4 — Analyse a star

Type any star name in the search box and click **RUN**:

```
TRAPPIST-1
TOI-700
HD 209458
LTT 9779
Kepler-452
```

The dashboard will show:
- Raw + detrended light curve plot
- BLS periodogram + phase-folded transit for each signal detected
- Batman transit model fit + residuals
- Stacked SNR, FAP proxy, classification confidence bars
- Planet radius, semi-major axis, habitable zone assessment
- All 4 class probabilities

---

### Step 5 — Run batch processing

To process a list of targets and generate a ranked candidate table:

```bash
python batch_runner.py
```

Or with a custom target list:

```bash
python batch_runner.py --targets my_targets.txt
```

Or for a quick test with only 50 stars:

```bash
python batch_runner.py --max 50
```

Output is saved to `results.json`. View it in the dashboard by clicking the **Batch Results** tab.

---

### Step 6 — Process a full TESS sector (optional)

```bash
python batch_sector.py --sector 1 --max_stars 200
```

This downloads all available light curves from TESS Sector 1 and runs the full pipeline on each star, outputting:
- `sector_1_results.csv` — all stars, ranked by planet probability
- `sector_1_candidates.csv` — only planet-classified signals with SNR > 4

## Feature Descriptions (17-feature vector)

| # | Feature | Description |
|---|---|---|
| 0 | `bls_power` | BLS peak power |
| 1 | `depth_ppm` | Transit depth in ppm |
| 2 | `duration_hrs` | Transit duration in hours |
| 3 | `snr_stacked` | Stacked SNR = (depth/noise) × √n_transits |
| 4 | `sec_ratio` | Secondary eclipse depth / primary depth (EB diagnostic) |
| 5 | `odd_even` | Odd vs even transit depth asymmetry (EB diagnostic) |
| 6 | `sin_corr` | Sinusoidal correlation (starspot diagnostic) |
| 7 | `duty_cycle` | Duration / period |
| 8 | `scatter_ppm` | Out-of-transit scatter in ppm |
| 9 | `pp_scatter` | Point-to-point scatter in ppm |
| 10 | `best_period` | BLS best period in days |
| 11 | `transit_frac` | Fraction of cadences in transit |
| 12 | `fap_proxy` | Peak power / 95th-percentile power (False Alarm Probability proxy) |
| 13 | `n_transits` | Number of expected transits in the baseline |
| 14 | `shape_score` | In-transit flux std / depth (V-shape vs flat-bottom) |
| 15 | `trend_power` | Long-term polynomial trend power in ppm |
| 16 | `depth_sig` | Depth / out-of-transit scatter |

## Evaluation Criteria vs Implementation

| PS Requirement | Implementation |
|---|---|
| Identify periodic dips | BLS periodogram, 3000 trial periods (0.5–20 days) |
| Classify: transit / eclipse / blend / other | Random Forest on 17 features, 4 classes |
| Apply classifier on science datasets | Batch runner + sector processor |
| SNR / significance levels | Stacked SNR = (depth/noise)×√n_transits; FAP proxy |
| Estimate period, depth, duration | BLS best-fit + batman model refinement |
| Visualize light curve + signal | 3 plots per star: LC, BLS+fold, batman+residuals |
| Confidence level | RF classifier probability + combined score |
| Multi-planet detection | Mask-and-repeat BLS up to 3 signals per star |

## Known Limitations

- The classifier was trained on 240 real samples (60 per class). The PS will provide a curated labeled dataset — load it using the "Load Training Data" button to immediately improve accuracy.
- Eclipsing binary training samples were gap-filled with synthetic data (ExoFOP has few EB dispositions). When the hackathon-provided EB labels arrive, retraining will improve significantly.
- Stellar mass from TIC metadata (`MASS` key) is often missing and defaults to 1.0 M☉, which affects the batman fit's semi-major axis estimate for non-solar stars.
